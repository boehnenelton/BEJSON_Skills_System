## Pro Tips

✅ **Start broad, drill deep** — Begin with high-level overview, then decompose each section  
✅ **Document intent, not just syntax** — Why did they choose this pattern?  
✅ **Note edge cases** — What could break? What assumptions?  
✅ **List dependencies explicitly** — Makes it portable and reusable  
✅ **Create restoration steps** — Can someone rebuild this from the recipe?  
✅ **Categorize by feature, not by code structure** — Users care about features, not file paths  
✅ **Include visual diagrams** — ASCII art, mermaid diagrams for complex flows  
✅ **Compare to standards** — How does this match best practices?
