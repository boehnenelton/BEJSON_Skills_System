# Writer Doc Architect: Agentic (AX-Standard)

Architect for 2026 agent-ready documentation stacks. Implements high-density, AX-optimized documentation including README.md, llms.txt, and AGENTS.md.

## Core Procedural Workflow

### Phase 1: Repository Discovery
1.  **Analyze Tech Stack**: Identify core languages, frameworks, and entry points.
2.  **Map Intent**: Determine the primary problem the codebase solves.
3.  **Audit Existing Docs**: Identify "token waste" and outdated patterns.

### Phase 2: Core Stack Deployment
1.  **Generate `llms.txt`**: Create a high-density, markdown-only index of the documentation tree.
2.  **Generate `AGENTS.md`**: Define behavioral rules, build paths, and the project's "Mental Model."
3.  **Generate `DESIGN.md`**: Encode OKLCH color tokens and visual rationale if applicable.

### Phase 3: AX-README Construction
1.  **Frontmatter Initialization**: Insert machine-readable YAML metadata at the top.
2.  **Semantic Chunking**: Structure the README using strict H2/H3 hierarchy for RAG efficiency.
3.  **One-Liner Integration**: Ensure a copy-pasteable installation command is prominent.
4.  **Agent Readiness Section**: Explicitly link to `llms.txt` and `AGENTS.md`.

### Phase 4: Visual & DX Polish
1.  **Token Alignment**: Ensure design tokens match the `DESIGN.md` specification.
2.  **Badge Auditing**: Remove "badge overload" in favor of dynamic trust signals.
3.  **Snippet Validation**: Ensure all code examples include imports and expected outputs.

### Phase 5: Verification
1.  **Token Efficiency Check**: Verify prose-to-fact ratio (Target: 20% prose, 80% facts).
2.  **Agent Simulation**: Verify if an AI agent can map the repo using only the generated files.

# Synergy & Awareness
- **Archival**: Consumes reports from `report-in-depth` or `report-concise`.
- **Styling**: Synergy with `css-becss` for styled, ARIA-compliant documentation.
- **Build Flow**: Integrated into the `skill-builder` workflow to provide immediate documentation for new skills.

## Reasoning Constraints
- **AX First**: Design for machine parsing; humans will benefit from the resulting clarity.
- **Fact Density**: Minimize "marketing fluff." Every sentence must have operational value.
- **RAG Optimization**: Use self-contained sections to prevent context loss during retrieval.
