# Post-Analysis Reporting: Concise Intelligence Briefing

This skill provides a high-efficiency protocol for transforming perceptual decomposition data into compact, high-density intelligence briefs. It prioritizes the "Signal-to-Noise Ratio" (SNR) and rapid cognitive comprehension.

## Core Procedural Workflow

### Phase 0: Pre-requisite Validation
1.  **Verify Analysis Baseline**: Confirm that an `analysis-perceptual-decomposition` analysis has been completed.
2.  **Verify Target Abstraction**: Confirm the user requires high-abstraction/executive summaries. If forensic depth is needed, transition to `post-analysis-reporting-in-depth`.

### Phase 1: Signal Distillation
1.  **Identify Primary Drivers**: Locate the 3-5 most critical emergent behaviors and structural patterns from the analysis data.
2.  **Filter Noise**: Remove granular atomic observations unless they are foundational to a primary driver.
3.  **Compress Logic**: Synthesize multi-step reasoning into direct, impact-oriented statements.

### Phase 2: Structural Briefing
1.  **Initialize Brief Structure**: Load the 5-section mandatory briefing skeleton from [references/briefing_protocol.md](references/briefing_protocol.md).
2.  **Populate High-Density Modules**:
    - **Executive Summary**: The "Bottom Line Up Front" (BLUF).
    - **System Topology**: High-level structural map.
    - **Primary Behaviors**: Core functional patterns.
    - **Anomalies & Risks**: Critical deviations.
    - **Strategic Conclusion**: Final conceptual synthesis.

### Phase 3: Cognitive Optimization
1.  **Apply Abbreviation Rules**: Use lists and bold keywords to facilitate rapid scanning.
2.  **Precision Language**: Use technical terminology only where it provides significant compression.
3.  **Visual Logic**: Ensure the Markdown layout guides the reader's eye to the most critical insights.

## Synergy & Awareness
- **Pre-requisite**: Always preceded by `analysis-perceptual-decomposition`.
- **Alternative**: Use `post-analysis-reporting-in-depth` for exhaustive forensic documentation.

## Reasoning Constraints
- **Signal Priority**: Never sacrifice clarity for brevity; only compress when it maintains or improves the signal.
- **Probabilistic Summarization**: Always include confidence levels for synthesized conclusions.
