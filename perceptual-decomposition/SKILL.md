# Perceptual Decomposition & Conceptual Pattern Recognition

This skill implements a rigorous, layered protocol for analyzing systems not as isolated objects, but as interacting fragments, relationships, and emergent behaviors.

## Core Procedural Workflow

Follow these phases sequentially. Do not collapse ambiguity or classify prematurely. For detailed guidance on each phase, consult [references/protocol_spec.md](references/protocol_spec.md).

### Phase 1: Atomic Observation (Decomposition)
1.  **Identify Boundaries**: Define the limits of every observable element.
2.  **Attribute Analysis**: List properties, behaviors, and relationships for each element.
3.  **Frequency & Syntax**: Detect repetition, spatial positioning, and context shifts.
4.  **Unitization**: Break large structures into interacting small structures.

### Phase 2: Local Pattern Detection (Interaction)
1.  **Neighbor Relationships**: Determine what influences or modifies each component.
2.  **Consistency Check**: Search for rhythm, clustering, dependencies, and echo patterns.
3.  **Emergent Meaning**: Allow meaning to arise from interaction patterns, not assumptions.

### Phase 3: Structural Mapping (Modeling)
1.  **Hierarchy Construction**: Create dependency chains and functional groupings.
2.  **Behavioral Regions**: Identify foundational vs. derivative parts and signal pathways.
3.  **System Characterization**: Determine if the system is ordered, adaptive, organic, or recursive.

### Phase 4: Pattern Convergence (Global Logic)
1.  **Mechanism Recognition**: Identify shared structural logic and consistent behavioral signatures.
2.  **Predictive Regularities**: Locate the governing architecture and organizing principles.
3.  **Uncertainty Representation**: Explicitly state where evidence is insufficient.

### Phase 5: Conceptual Awareness (Synthesis)
1.  **Identity Formation**: Answer what the system is attempting to do and what hidden assumptions organize it.
2.  **Emergence Analysis**: Identify properties arising from multi-scale interaction.

## Reporting Handoff Protocol
Once Phase 5 is complete and a stable conceptual awareness model is established, you MUST evaluate the need for formal documentation:
- **High-Density / Forensic / Detailed**: Transition to `post-analysis-reporting`.
- **High-Abstraction / Executive / Brief**: Transition to `concise-analysis-reporting`.

## Resource Navigation

## Reasoning Constraints

- **Recursive Analysis**: Analyze parts -> relationships -> patterns -> systems; repeat until stable.
- **Intent Recognition**: Infer intent probabilistically through optimization behavior and resource allocation.
- **Anti-Compression**: Never rely on first impressions or discard anomalies without explanation.

## Final Output Structure

Every analysis must conclude with:
1.  **Atomic Observations**
2.  **Local Patterns**
3.  **Structural Relationships**
4.  **System-Level Behaviors**
5.  **Emergent Properties**
6.  **Probable Intent**
7.  **Confidence Estimates**
8.  **Unresolved Ambiguities**
9.  **Alternative Interpretations**
10. **Final Conceptual Awareness Model**
